## 2. GIM 文件结构

GIM 文件本质上是一个压缩包，解压后包含以下四个顶层目录：

```
XX变电站新建.gim/
├── CBM/     # 工程层级结构文件（.cbm / .fam）
├── DEV/     # 设备定义与IFC文件（.dev / .fam / .ifc）
├── PHM/     # 组合模型文件（.phm）
└── MOD/     # 基础几何模型文件（.mod）
```

### 各目录职责

| 目录 | 文件类型 | 数量级 | 职责 |
|------|----------|--------|------|
| CBM/ | `.cbm`, `.fam` | 数百个 | 构建工程的层级树状结构，从工程根节点到设备级节点 |
| DEV/ | `.dev`, `.fam`, `.ifc` | 数百个 | 描述具体设备的属性与参数，IFC文件承载三维建筑信息模型 |
| PHM/ | `.phm` | 大量 | 组合模型，将多个MOD组装为可复用的设备模型 |
| MOD/ | `.mod` | 大量 | 基础几何模型，定义最底层的几何形状与材质 |

## 3. 层级结构分析

### 3.1 入口文件 `project.cbm`

```
BLHA=<纬度>,<经度>,<海拔>,<方向角>
SUBSYSTEM=<UUID>.cbm
SCH=project.sch
TYPE=TS
```

| 字段 | 含义 |
|------|------|
| `BLHA` | 工程基准坐标（纬度、经度、海拔、方向角） |
| `SUBSYSTEM` | 一级子系统引用（UUID命名的CBM文件） |
| `SCH` | 逻辑模型引用（project.sch） |
| `TYPE` | 工程类型标识 |

### 3.2 一级子系统

文件：`<UUID>.cbm`

```
ENTITYNAME=F1System
BASEFAMILY=<UUID>.fam
SUBSYSTEMS.NUM=14
SUBSYSTEM0=<UUID>.cbm
...（共14个子系统）
IFC.NUM=12
IFC0=电气二次0317其他.ifc
...（共12个IFC文件）
```

| 字段 | 含义 |
|------|------|
| `ENTITYNAME` | 实体名称（F1System） |
| `BASEFAMILY` | 基础族文件引用 |
| `SUBSYSTEMS.NUM` | 子系统数量 |
| `SUBSYSTEM0~N` | 二级子系统CBM文件引用 |
| `IFC.NUM` | IFC文件数量 |
| `IFC0~N` | IFC文件引用（位于DEV目录） |

### 3.3 层级结构示意

```
project.cbm（工程根）
└── 一级子系统.cbm（F1System）
    ├── 二级子系统0.cbm
    │   ├── 三级子系统.cbm
    │   │   └── ... → 设备级.cbm
    │   └── ...
    ├── 二级子系统1.cbm
    │   └── ...
    └── ...（共14个二级子系统）
```

## 5. 逻辑模型分析

### 5.1 引用链

```
project.cbm → SCH=project.sch → zjx.std + zjx.sld
```

| 文件 | 格式 | 职责 |
|------|------|------|
| `project.sch` | 文本 | 逻辑模型入口，引用 std 和 sld 文件 |
| `zjx.std` | XML | 变电站逻辑模型定义，定义电压等级、间隔和设备 |
| `zjx.sld` | SVG | 主接线图，包含各间隔的电气元件符号和连接关系 |

### 5.2 逻辑模型内容（zjx.std）

**电压等级**：220kV（标识为 AE）

**间隔（Bay）定义**：

| 间隔标识 | 间隔名称 | 说明 |
|----------|----------|------|
| C2 | 7E出线间隔 | 组合电器GIS |
| E3 | 母联II | 母联断路器间隔 |
| E4 | 母联I | 母联断路器间隔 |
| A240 | 2号备用主变本体系统 | 主变压器间隔 |

**设备类型**：

| 类型代码 | 设备名称 |
|----------|----------|
| SAR | 避雷器 |
| DIS | 隔离开关 |
| VTR | 电压互感器 |
| CTR | 电流互感器 |
| PTR | 变压器 |
| EVPIS | 高压带电显示器 |

## 6. 文件引用关系

### 6.1 核心引用链

```
project.cbm
  │
  ├──→ 一级子系统.cbm
  │      ├──→ 二级子系统.cbm → ... → 设备级.cbm
  │      ├──→ IFC0~IFCN → *.ifc（DEV目录）
  │      └──→ SCH → project.sch → *.std + *.sld
  │
  └──→ 设备级.cbm
         └──→ OBJECTMODELPOINTER → *.dev → *.phm → *.mod
```

### 6.2 引用关系说明

| 引用路径 | 说明 |
|----------|------|
| `project.cbm → SUBSYSTEM → *.cbm` | 工程根到子系统的层级引用 |
| `子系统.cbm → IFC0~N → *.ifc` | 子系统到IFC建筑信息模型的引用 |
| `子系统.cbm → SCH → project.sch` | 子系统到逻辑模型的引用 |
| `设备级.cbm → OBJECTMODELPOINTER → *.dev` | 设备到设备定义文件的引用 |
| `*.dev → *.phm` | 设备定义到组合模型的引用 |
| `*.phm → *.mod` | 组合模型到基础几何模型的引用 |

### 6.3 文件-设备关系（FileDevRelation.cbm）

该文件记录了 IFC 文件与设备的对应关系：

- 共记录 **24** 个 IFC 文件与设备的映射
- 每个 IFC 文件包含多个设备（CBM引用）
- 例如"电气二次0317"包含 **112** 个设备
