# FAM 文件格式

## 文件概述

FAM（Family/Attribute）文件是 GIM 工程中的属性定义文件，采用键值对文本格式。FAM 文件本身**不包含** `BASEFAMILY` 字段，不存在属性继承链——`BASEFAMILY` 字段出现在 CBM 和 DEV 文件中，用于**指向** FAM 文件，建立单向引用关系。

FAM 文件根据所处目录不同，其内容和组织方式有显著差异：

| 所在目录     | 变电工程                                       | 线路工程                                       |
| ------------ | ---------------------------------------------- | ---------------------------------------------- |
| **CBM 目录** | 75.6% 为空文件，非空文件含建筑/结构或设备参数 | 全部非空，含导线/地线参数或物资信息             |
| **DEV 目录** | 全部非空，统一使用 `[设计参数]` 节             | 全部非空，含详细线路设备参数（绝缘子串/基础等） |

## 文件格式

- **编码**：UTF-8
- **行分隔符**：换行符
- **节标题**：INI 风格的 `[节名]`（仅变电工程部分 FAM 使用）
- **空值表示**：线路工程用空字符串，变电工程用 `/` 或 `-`

### 三种行格式

FAM 文件存在三种行格式，因工程类型和目录位置而异：

| 格式           | 模式                    | 适用范围                                       |
| -------------- | ----------------------- | ---------------------------------------------- |
| 三段式（中英） | `中文标签=英文KEY=值`   | 线路工程全部 FAM；变电工程 CBM 中 317 个非节文件 |
| 三段式（中中） | `中文KEY=中文KEY=值`    | 变电工程含 `[设计参数]` 节的 FAM                |
| 节标题         | `[设计参数]`            | 变电工程部分 CBM FAM 和全部 DEV FAM            |

> **解析建议**：三段式行可通过两个 `=` 分隔符解析为 `label`/`key`/`value` 三元组。`label` 用于界面展示，`key` 适合进入字段字典，`value` 为原始值。

## 字段说明

### 通用特性

FAM 文件**不包含** `BASEFAMILY` 字段。FAM 文件是被引用方，引用关系由 CBM 和 DEV 文件建立：

```text
CBM 文件 ──BASEFAMILY──→ <uuid>.fam（CBM 目录下的 FAM）
DEV 文件 ──BASEFAMILY──→ <uuid>.fam（DEV 目录下的 FAM）
```

### 变电工程节标题

变电工程部分 FAM 文件使用 INI 风格的节标题：

| 节标题       | 说明                           |
| ------------ | ------------------------------ |
| `[设计参数]` | 设计阶段参数（实测中仅此一种） |

> **修正说明**：早期文档列出 `[产品参数]`/`[施工参数]`/`[测试参数]`/`[运检参数]` 四种节，实际 demo-substation 数据中均未出现。变电工程 FAM 文件只使用 `[设计参数]` 节。

## 引用关系

```text
CBM 文件
└── BASEFAMILY → <uuid>.fam（CBM 目录下的 FAM，单向引用，无继承链）
└── BASEFAMILY1..N → <uuid>.fam（仅变电 F3System 多 FAM 引用）

DEV 文件
└── BASEFAMILY → <uuid>.fam（DEV 目录下的 FAM，单向引用，无继承链）
```

FAM 文件内部不再引用其他 FAM 文件，属性继承链不存在。CBM/DEV 通过 `BASEFAMILY` 字段单向指向 FAM 文件获取属性。

## 统计数据

### 按目录和工程类型统计

| 位置                       | 总数  | 空文件 | 有节标题 | 无节标题（非空） |
| -------------------------- | ----: | -----: | -------: | ---------------: |
| demo-line CBM FAM          | 21967 |      0 |        0 |            21967 |
| demo-line DEV FAM          |  4518 |      0 |        0 |             4518 |
| demo-substation CBM FAM    |  8877 |   6710 |     1850 |              317 |
| demo-substation DEV FAM    |  4179 |      0 |     4179 |                0 |

### 关键发现

- **线路工程**：所有 FAM 文件（CBM 和 DEV）均非空，使用三段式（中英）格式，无节标题。
- **变电工程 CBM FAM**：75.6% 为空文件（6710/8877），非空文件中 85.4% 使用 `[设计参数]` 节（1850 个），14.6% 无节标题（317 个，多为建筑/结构参数）。
- **变电工程 DEV FAM**：全部非空，统一使用 `[设计参数]` 节 + 三段式（中中）格式。

## 变电工程与线路工程的差异

| 维度         | 变电工程                                       | 线路工程                                       |
| ------------ | ---------------------------------------------- | ---------------------------------------------- |
| 行格式       | 三段式（中中）或三段式（中英，仅 CBM 317 个）  | 三段式（中英）                                 |
| 节标题       | `[设计参数]`（DEV 全部，CBM 24.4% 非空文件中 85.4%） | 无                                             |
| 空值表示     | `/` 或 `-`                                     | 空字符串                                       |
| CBM FAM 空文件率 | 75.6%（6710/8877）                         | 0%                                             |
| DEV FAM 内容 | 设备电气参数（额定电压/电流/型号等）           | 线路设备参数（绝缘子串/基础/导线力学参数等）  |
| KEY 风格     | 中文重复（`生产厂家=生产厂家=`）或英文（建筑类）| 英文（`MANUFACTURER=`/`VOLTAGE=`）             |

## 解析建议

| 元素   | 说明                          |
| ------ | ----------------------------- |
| label  | 第一个 `=` 前的中文标签       |
| key    | 第二个字段中的 KEY            |
| value  | 第二个 `=` 后的原始值         |
| section| `[节名]` 行，适用于变电工程   |

解析时需注意：
- 三段式行的 value 可能为空（线路）或 `/`/`-`（变电），均应视为"未设置"。
- 变电工程含节标题的 FAM 文件，节标题后的行均属于该节，直到下一个节标题或文件结束。
- 线路工程 FAM 文件无节标题，所有行均为顶级键值对。
- 空文件（变电 CBM 6710 个）应直接跳过，不尝试解析行内容。
